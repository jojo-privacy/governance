# Maintainers

This document defines the roles, responsibilities, and expectations of maintainers within the Jojo Privacy organization.

Maintainers are entrusted with safeguarding the security, privacy, and integrity of the project.

---

## Roles

### Project Lead
The Project Lead:
- Has final decision authority
- Defines project direction
- Approves maintainers
- Oversees security-critical decisions

### Maintainers
Maintainers:
- Review and approve pull requests
- Enforce contribution and security standards
- Participate in technical discussions
- Protect the projectâ€™s privacy guarantees

### Contributors
Contributors:
- Propose changes via pull requests
- Participate in discussions
- Do not have merge authority

---

## Maintainer Responsibilities

All maintainers must:
- Review code carefully and critically
- Prioritize security and privacy implications
- Require tests for behavioral changes
- Reject unclear or unsafe changes
- Avoid merging changes they do not fully understand

Maintainers must **not**:
- Merge code without review
- Approve cryptographic changes casually
- Trade security for speed

---

## Adding a Maintainer

A new maintainer may be added if they:
- Have a consistent history of high-quality contributions
- Demonstrate strong understanding of security and privacy principles
- Are trusted to act responsibly and transparently

Final approval is required from the Project Lead.

---

## Removing a Maintainer

A maintainer may be removed if they:
- Violate project principles
- Act negligently in security matters
- Become inactive for an extended period
- Breach trust or act in bad faith

Removal decisions prioritize project safety over individual status.

---

## Conflict of Interest

Maintainers must disclose any conflicts of interest that could affect decisions, including:
- Commercial involvement
- Employment related to competing products
- External pressure to prioritize features over security
